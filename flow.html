AI 接單解決方案

讓 AI 接住每一個詢單
24/7 自動引導，15 分鐘完成報價

以 Asgard AI 的 Odin 工作流引擎 為核心，搭配客製化前後端與基本資料層，將散客詢價流程從「0.5～3 天的來回溝通」壓縮至即時完成。

核心技術
Odin 工作流 ＋ 客製化交付
本次 POC 以 Asgard Odin 工作流引擎為編排核心，搭配客製化前後端與基本 DB / AP Server 完成交付。Sindri Agent Hub、Mimir Data Insight 等進階模組為 Phase 2 未來擴展。
Asgard 平台
⚙️
Odin
Asgard Studio · 零程式碼 AI 工作流
  • 對話流程編排（接單主流程、條件分支、轉接邏輯）
  • LLM 節點（意圖識別、欄位缺口偵測、話術生成）
  • 串接 Vision / Image Gen API（圖檔解析、模擬圖）
  • Webhook 入口（網頁、LINE 雙管道）
客製化開發
🛠️
前後端 + DB / AP Server
由 Asgard AI 團隊客製化交付
  • 嵌入式 Chatbot 前端（React，可內嵌至官網）
  • 後端 API（Session、訂單草稿、舊客查詢、通知）
  • 基本 DB（訂單草稿、客戶資料、對話紀錄）
  • AP Server 部署 + 基本監控
完整流程
從詢問到訂單成立
AI 全程引導，不漏接任何一個詢單。以下為完整接單流程，標示各步驟由哪個 Asgard AI 模組負責執行。
入
客戶透過 LINE / 網頁發出詢問
不限管道、不限時間。散客發出任何形式的詢問訊息（報價、補印、規格詢問均可），系統 24/7 即時接線，不再因為非上班時間而漏接。
Odin · Webhook 入口 客製化後端 · Session 建立
分
意圖識別 — 詢價 / 補印 / 訂單查詢
AI 自動判斷客戶意圖，分流至對應的處理路徑。無論客戶說「我要印貼紙」或「上次那個可以再來一次嗎」，都能正確識別並引導。
Odin · 路由器 Odin · Intake 節點
🆕 新客詢價
建立新 Session，從第一個欄位開始引導收集。AI 全程使用白話語言，不出現印刷術語。
🔄 舊客補印
以 Email / 電話比對歷史訂單，自動帶入所有舊規格，只確認差異欄位，最快 2 問完成。
收
欄位缺口偵測 — 白話引導收集規格
系統核心：AI 持續追蹤 14 個必填欄位的收集狀態，每輪對話後判斷哪個欄位最重要、還缺什麼，自動生成下一個白話追問，不重複、不遺漏。
Odin · 欄位偵測節點 客製化 DB · 材質知識
材
材質智能對應 — 限定官網 8 種材質
客戶說「要那種有金屬感的」，AI 自動對應至「亮銀龍」或「反銀龍」並附上說明。所有材質選項嚴格限定為官網提供的 8 種，確保可生產性，不接受超出範圍的需求。
Odin · 材質推薦節點
圖
設計稿解析 + AI 貼標模擬圖生成
客戶上傳設計稿，AI 自動解析色數與特殊工藝（燙金、打凸等）。同時依據材質、容器類型與使用場景，生成貼標於實際產品上的情境模擬圖供客戶預覽。
LLM Vision · 圖檔解析 Odin · Image Gen 節點
情
情緒偵測 — 負面累積自動轉真人
AI 持續偵測對話中的情緒訊號。當客戶出現連續不滿、要求議價或超出 AI 知識範圍時，自動觸發通知，將完整對話紀錄交給業務人員接手——客戶無需重複說一遍。
Odin · 情緒偵測節點 Odin · 轉接 Email 通知
單
訂單草稿整合確認
所有欄位收集完成後，AI 匯整成結構化訂單草稿，以視覺化方式呈現給客戶確認，包含品名、尺寸、數量、材質、交期等所有必要資訊。
Odin · 訂單總結節點
存
寫入資料庫 + 即時通知業務
訂單草稿自動寫入客製化 DB，同步觸發 Email / LINE 通知業務人員。業務收到完整的結構化資料，無需再次電話確認規格，直接進行報價。
客製化 DB · 訂單寫入 Odin · 業務通知
✓
業務確認報價 → 訂單正式成立
業務人員依據完整的訂單草稿進行報價確認。標準化規格的訂單未來可由 AI 直接完成報價，僅在金額超過門檻或規格超出範圍時才需人工介入。
✅ 訂單成立
