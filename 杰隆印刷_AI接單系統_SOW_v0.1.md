# 杰隆印刷 × Asgard AI
## AI 自動接單系統　工作說明書（SOW, Statement of Work）

版本：v0.1  
日期：2026/05/27

---

## 一、專案簡介

本專案旨在為杰隆印刷有限公司導入一套 AI 自動接單系統，以 Asgard AI 平台的 **Odin 工作流引擎** 為核心，搭配客製化前後端與基本 DB / AP Server，建立可 24 小時全天候運作的智慧詢價與下單機制。

系統將涵蓋「客戶詢價 → AI 引導收集規格 → 訂單草稿生成 → 業務確認成立」之完整流程，讓中小型客戶無需具備印刷專業知識，即可透過白話對話自助完成詢價作業，取代目前仰賴人力即時回應的 Email / LINE 溝通模式。

---

## 二、專案目標與效益

本專案目標在於解決現有詢價流程的斷點，並達成以下效益：

- **縮短詢價週期**：從現況的 0.5～3 天，縮短至 15 分鐘以內完成資訊收集。
- **全天候接單**：非上班時間與假日由 AI 自動接線，跨時區客群全數覆蓋。
- **降低業務重複工作**：AI 完整收集規格後再移交業務確認，減少來回溝通次數。
- **提升回頭客效率**：舊客補印同款僅需 ≤ 3 輪對話即可完成確認。
- **訂單資訊標準化**：以結構化 JSON 輸出訂單草稿，可銜接現有訂單追蹤 Excel。

## 三、主要功能需求

本專案系統分為 **網頁 Chatbot（客戶端）**、**Odin 工作流（後端）**、**資料層（AP Server / DB）** 以及 **管理後台（業務端）** 四大部分。Phase 2 擴充功能另行標注。

| 平台 | 功能 | 子功能 | 功能說明 |
|------|------|--------|---------|
| **網頁 Chatbot（客戶端）** | 對話入口 | 網頁嵌入框 | 可嵌入杰隆官網的對話介面，支援桌面與行動裝置瀏覽 |
| **網頁 Chatbot（客戶端）** | 對話入口 | 意圖識別 | 識別客戶來意（詢價 / 訂單追蹤 / 舊客補印 / 閒聊 / 轉接），導入對應對話流 |
| **網頁 Chatbot（客戶端）** | 詢價流程 | 開場引導 | 以白話問候，不使用印刷術語，建立 Session 並判斷新/舊客身份 |
| **網頁 Chatbot（客戶端）** | 詢價流程 | 欄位缺口偵測 | 即時追蹤 A 類 14 項必問欄位的收集進度，依優先序追問尚未填寫的欄位 |
| **網頁 Chatbot（客戶端）** | 詢價流程 | 材質智能引導 | 客戶白話描述（如「有光澤感」、「銀色金屬感」）自動對應官網 8 種材質，不推薦目錄外材料 |
| **網頁 Chatbot（客戶端）** | 詢價流程 | 條件欄位觸發 | 特定回答（如選擇成捲、提到金色反光）自動追問對應 B 類條件欄位（捲向、燙金規格等） |
| **網頁 Chatbot（客戶端）** | 詢價流程 | 訂單草稿確認 | A 類欄位全部收集後彙整並以條列式顯示訂單摘要，供客戶最終確認 |
| **網頁 Chatbot（客戶端）** | 圖檔上傳 | 設計稿上傳 | 支援客戶上傳設計檔（JPG / PNG / PDF），觸發 AI 圖像分析 |
| **網頁 Chatbot（客戶端）** | 圖檔上傳 | AI 圖像分析 | LLM Vision 解析設計稿，自動判斷色數、有無特殊工藝（燙金、打凸）需求並回報客戶確認 |
| **網頁 Chatbot（客戶端）** | AI 模擬圖 | 情境圖生成 | 依收集欄位（尺寸、材質、容器類型）生成貼標模擬情境圖，在 A 類欄位收集 50%+ 時主動詢問 |
| **網頁 Chatbot（客戶端）** | AI 模擬圖 | 模擬圖預覽 | 在對話框內顯示低解析度預覽圖（含浮水印），說明非正式打樣 |
| **網頁 Chatbot（客戶端）** | 舊客識別 | Email / 電話比對 | 客戶輸入聯繫方式後，自動比對客戶資料庫，判斷是否為舊客 |
| **網頁 Chatbot（客戶端）** | 舊客識別 | 歷史訂單帶入 | 一年內舊客帶入上次訂單欄位，僅確認差異；超過一年則重新確認全部欄位 |
| **網頁 Chatbot（客戶端）** | 補印流程 | 補印舊款式 | 舊客選擇「補印同款」時，帶入舊資料，確認數量與出貨日期即可完成 |
| **網頁 Chatbot（客戶端）** | 情緒偵測 | 負面情緒判斷 | 連續 2 次偵測到明確不滿語句時，切換至轉接模式 |
| **網頁 Chatbot（客戶端）** | 情緒偵測 | 真人轉接通知 | 觸發後停止推進欄位收集，保留完整對話紀錄，發送通知給業務人員 |
| **網頁 Chatbot（客戶端）** | 情緒偵測 | 超規格轉接 | 需求超出官網 8 種材質、尺寸超出生產範圍，或客戶明確要求真人時，自動轉接 |
| **網頁 Chatbot（客戶端）** | 多語言 | 英文詢價 | 偵測英文輸入，以英文回應並將欄位對應至中文訂單格式 |
| **Odin 工作流（後端）** | 工作流引擎 | Session 管理 | 每筆對話建立獨立 Session，跨輪追蹤對話狀態與欄位填寫進度 |
| **Odin 工作流（後端）** | 工作流引擎 | 對話節點編排 | 以 Odin 零程式碼工作流設計完整詢價路徑、條件分支邏輯與轉接觸發規則 |
| **Odin 工作流（後端）** | 工作流引擎 | 圖檔解析串接 | 對接 LLM Vision API，解析客戶上傳的設計稿圖像 |
| **Odin 工作流（後端）** | 工作流引擎 | 模擬圖生成串接 | 對接 Image Gen API，依訂單欄位生成貼標情境圖 |
| **Odin 工作流（後端）** | 通知機制 | Email 通知 | 訂單草稿完成或觸發人工轉接時，自動發送 Email 通知業務人員 |
| **Odin 工作流（後端）** | 通知機制 | LINE 通知 | *(Phase 2)* 透過 LINE Webhook 推送通知至業務群組 |
| **資料層（AP Server / DB）** | 訂單管理 | 訂單草稿儲存 | 以結構化 JSON 儲存對話完成後的訂單草稿，欄位格式對應現有訂單追蹤 Excel |
| **資料層（AP Server / DB）** | 客戶資料 | 客戶資料庫 | 以 Email / 電話為索引儲存客戶基本資料與歷史訂單，支援舊客識別查詢 |
| **資料層（AP Server / DB）** | 對話紀錄 | 完整 Log 保存 | 儲存每筆對話完整紀錄（含時間戳），供品質分析與後續訓練使用 |
| **資料層（AP Server / DB）** | 系統管理 | AP Server / 容器化 | 後端 API 服務、Session 管理、容器化部署與基本運行監控 |
| **管理後台（業務端）** | 訂單確認 | 草稿列表 | 業務人員可查看所有已完成詢價的訂單草稿清單，含填寫完整度指標 |
| **管理後台（業務端）** | 訂單確認 | 確認 / 備註 | 業務人員可確認訂單草稿、補充備註或退回，確認後觸發通知客戶 |
| **管理後台（業務端）** | 轉接管理 | 人工接管紀錄 | 顯示觸發情緒偵測或超規格轉接的對話紀錄清單，供業務後續跟進 |
| **LINE 官方帳號（Phase 2）** | 接收管道 | LINE Webhook | *(Phase 2)* 整合 LINE 官方帳號，統一接收 LINE 頻道詢價進入 Odin 工作流 |
| **報價引擎（Phase 2）** | 自動報價 | 即時報價計算 | *(Phase 2)* 依杰隆提供的定價邏輯，完成欄位收集後即時輸出報價金額 |

## 四、驗收與成功指標

### 4.1 功能驗收項目

| 驗收項目 | 通過標準 |
|---------|---------|
| 完整詢價對話流 | AI 能從零引導收集完 A 類全部 14 項欄位，無需客戶主動說明印刷術語 |
| 材質約束 | AI 在材質選擇時只推薦官網 8 種材質，不出現其他材料名稱 |
| 舊客識別 | 輸入已知 Email 或電話後，正確帶入歷史訂單資料 |
| 補印流程 | 舊客選擇補印同款，3 輪對話內完成確認 |
| 模擬圖生成 | 上傳設計稿並提供容器類型後，30 秒內輸出可預覽模擬圖 |
| 情緒轉接 | 模擬連續 2 次負面情緒輸入，正確觸發業務通知 Email |
| 訂單草稿輸出 | 對話完成後輸出符合 Excel 欄位格式的結構化 JSON |
| 多語言 | 輸入英文詢價，AI 正確理解並以英文回應，欄位對應中文訂單格式 |

### 4.2 非功能驗收項目

| 驗收項目 | 標準 |
|---------|------|
| 回應時間 | 每輪對話回應 < 3 秒（不含模擬圖生成） |
| 個資保護 | 收件人姓名、地址、電話欄位不在對話紀錄中明文顯示 |
| 對話長度 | 新客完成所有 A 類欄位收集 ≤ 15 輪對話 |
| 回頭客效率 | 補印同款 ≤ 3 輪對話完成確認 |
| 裝置相容 | 對話介面在 iOS Safari、Android Chrome、桌面 Chrome 正常顯示 |
